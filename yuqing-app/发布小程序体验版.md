# 微信小程序体验版（测试版）发布指南

本项目 **H5 与小程序共用同一套源码**（`src/` 下页面与组件），改一处即同步到两端。发布前请先确认 H5 与小程序都已按需构建，内容一致。

## 一、前置条件

1. **微信开发者工具**：已安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
2. **小程序 AppID**（正式体验版必填）：
   - 登录 [微信公众平台](https://mp.weixin.qq.com/) → 小程序 → 开发管理 → 开发设置 → AppID。
   - 若仅本地预览，可使用当前配置的 `touristappid`（游客模式）；**上传体验版需使用自己的 AppID**。

## 二、填写 AppID（发布体验版时必做）

在项目根目录的 `project.config.json` 中，将 `appid` 改为你的小程序 AppID：

```json
{
  "appid": "你的AppID",
  ...
}
```

改完后重新执行一次构建（见下一步），再上传。

## 三、构建（与 H5 内容同步）

在项目根目录执行，按需选择：

```bash
# 只发小程序：构建微信小程序，产物在 dist/weapp
npm run release:weapp
# 或
npm run build:weapp
```

```bash
# 同时发 H5 + 小程序：先构建 H5，再构建小程序，两端内容一致
npm run build:h5
npm run build:weapp
```

源码一致，构建产物分别位于 **`dist/h5`**（H5）和 **`dist/weapp`**（小程序）。

## 四、用微信开发者工具上传体验版

1. 打开 **微信开发者工具**。
2. **导入项目**：
   - 选择「导入项目」或「+」；
   - **项目目录**：选择本项目的 **`dist/weapp`** 文件夹（即 `yuqing-app/dist/weapp`）；
   - **AppID**：若已在 `project.config.json` 中填写，会自动带出；否则在工具里选择或填写你的 AppID；
   - 点击「导入」。
3. 在开发者工具中确认能正常编译、预览无报错。
4. **上传为体验版**：
   - 顶部菜单选择 **「上传」**；
   - 填写**版本号**（如 `1.0.0`）和**项目备注**（如「测试版-舆情分析」）；
   - 点击「上传」。
5. 上传成功后，登录 **微信公众平台** → **版本管理**：
   - 在「开发版本」中找到刚上传的版本；
   - 可设为 **「选为体验版」**，体验版二维码即可给测试人员扫码使用。

## 五、测试人员使用体验版

- 在公众平台 **版本管理** 中，对当前体验版可 **下载体验版二维码**。
- 测试人员用微信扫码即可打开小程序体验版（需在公众平台「成员管理」中配置为项目成员，或使用「体验成员」）。

## 六、常见问题

| 问题 | 处理 |
|------|------|
| 上传时提示 AppID 无效 | 在 `project.config.json` 中填写正确的 AppID 后重新 `npm run build:weapp`，再导入 `dist/weapp` 上传。 |
| 导入后报错或白屏 | 确认导入的是 **`dist/weapp`** 目录，且已完整执行过 `npm run build:weapp`。 |
| 体验版看不到 | 在公众平台「版本管理」中把刚上传的开发版本「选为体验版」。 |

---

**快速流程小结**：改 AppID → `npm run release:weapp` → 开发者工具导入 `dist/weapp` → 上传 → 公众平台选为体验版。
